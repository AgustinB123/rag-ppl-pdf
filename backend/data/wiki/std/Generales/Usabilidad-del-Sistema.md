### Introducción {: #Intro} 
**Objetivo** {: #Obj}

El presente manual es una introducción a la funcionalidad básica de acceso y de
usabilidad del sistema FPA Portfolio.

**Alcance del sistema** {: #Alc} 

El Sistema contempla todos los procesos necesarios para la administración
Tesorería.

**Descripción básica del sistema** {: #descr} 

**Barra de Menú**
El Menú principal de FPA está formado por las siguientes secciones:

**Archivo:** Desde aquí el usuario podrá administrar, ajustándose a los permisos que
posea, las altas, bajas o modificaciones que desee efectuar sobre las principales
entidades del Sistema.

**Operaciones:** Podemos visualizar, cargar o modificar operaciones. También
podemos supervisar las distintas instancias por las que pasa una operación
(workflow de operaciones)

**Transacciones:** En esta sección podemos cargar, modificar o visualizar
transacciones, órdenes y minutas de bolsa. También podemos supervisar las
distintas instancias por las que pasa una transacción.

**Informes:** Contiene todos los informes disponibles en el sistema. Los informes
contenidos en el ítem Auxiliar son componentes de otros informes.

**Eventos:** Contiene todos los eventos disponibles del sistema. Los eventos
contenidos en el ítem Auxiliar son componentes de otros eventos.

**Utilitarios:** Permite definir datos relativos a la impresión, consultar procesos en
ejecución o errores que hayan surgido, cambiar la fecha del sistema, realizar la
apertura y cierre del día, configurar la seguridad del sistema, generar la agenda de
cupones entre otras funcionalidades.

**Herramientas:** Permite acceder a las opciones Escritorio Inteligente, Menú de
Últimas Acciones y Mensajería Instantánea.

**Tipos de Mensajes**  { : #Tipo} 

El sistema permite enviar mensajes con las siguientes categorías:
 * Información (Azul)
 * Advertencia (Amarillo)
 * Alerta (Rojo)
 * Operación (Verde)

A continuación se visualiza un ejemplo de cada tipo de mensaje y cómo se
selecciona el Tipo:

![tiposdemsj.png](/tiposdemsj.png)

Tras seleccionar el Tipo (Información, Advertencia, Alerta, Operación) se escribe
el contenido y se envía presionando la opción enviar (Alt + Enter)
Los mensajes del tipo Operación están vinculadas a una operación del sistema.
Dentro del mensaje contiene el NrOperacion de la misma, anteponiendo el
símbolo #.

Existe la posibilidad de filtrar los mensajes por Tipo, se utiliza el filtro como se
visualiza a continuación:

![filtrarportipo.png](/filtrarportipo.png)

Al escribir un mensaje, se puede mencionar a otros usuarios anteponiendo el
carácter espacial **@** y el sistema como ayuda muestra el listado de usuarios para
su selección.

![etiquetarusuario.png](/etiquetarusuario.png)

Al mencionar un usuario, le llega una **notificación** de escritorio sin importar que
tenga deshabilitadas las alertas.

![mensajenoleido.png](/mensajenoleido.png)

Al enviarse un mensaje, se emite con un estado **No leído.**
El usuario que lo recibe, puede marcarlo y desmarcarlo de forma manual como
**Leído / No leído**.

![marcarcomonoleido.png](/marcarcomonoleido.png)

La lectura de un mensaje enviado se indica con **Tildes Azules**.

![visto.png](/visto.png)

Por cada mensaje enviado, es posible consultar **quienes leyeron el mensaje
enviado** posicionándose sobre la opción Visto por .

![vistoporbrenda.png](/vistoporbrenda.png)

En caso de necesitar borrar un mensaje, sólo se podrán **borrar los mensajes
propios** siempre y cuando no hayan sido leídos por otros usuarios. También se
puede configurar un tiempo máximo hasta el cual se pueden borrar. El **borrado
es lógico**, por lo que no se pierden esos mensajes borrados.

**Canales** {: #Chan} 

Desde el ABM de canales, los usuarios que tengan permisos podrán crear canales
de difusión. Un canal consiste en un Nombre de Canal y su Descripción.
Cada mensaje puede pertenecer a un canales específico. Si no se crean canales
se utiliza por default la opción Todos los canales.
Los mensajes que no se encuentran en ningún canal, son &quot;globales&quot;.
Según el perfil de usuario un canal puede ser: Obligatorio, Restringido u Opcional.
En caso de ser opcional, el usuario puede suscribirse de forma manual en el
diálogo de preferencias de usuario.

![canales.png](/canales.png)

**Suscripción a canales**

Desde el Menú Principal\Utilitarios\Preferencias se accede a la pantalla de
configuración de Mensajes.

![suscripcionacanales.png](/suscripcionacanales.png)

Desde esta opción se puede configurar a cuál/es canal/es de mensajes se
suscribe un usuario.

**Nota:** esta funcionalidad se encuentra disponible a partir de la versión 6.6.

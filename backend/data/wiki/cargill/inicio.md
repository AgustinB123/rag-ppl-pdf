## Configuración regional
Cargill debe usar la configuración regional Inglés (Estados Unidos) debido a que usa un formato desde el Core de la forma "dd-MMM-yyyy", la cual si se usa con formato regional en español, le agrega puntos a los meses y genera errores en las consultas a la base de datos Oracle.
